# Finder

Finder is a simple search engine with an AI component called Find4U. You can try it out [here](https://finder-locw.onrender.com).

## How do I run it?

To run it, just run:
``` sh
make
```
with:

- Node.js 26
- Python 3.14

installed. This will start the web server.

If this is your first time running `make`, it might say:
```
Makefile:17: No Makefile.secret detected. You can choose to create one using the Makefile.secret rule. Or you can make it yourself.
No Makefile.secret found: run make Makefile.secret
make: *** [Makefile:46: run] Error 1
```
The solution is to run `make Makefile.secret`. It will install JS dependencies in their respective `node_modules` folders and Python dependencies in a virtual environment called `env` before prompting you for your OpenRouter API key:
```
Enter your OpenRouter API key:
```
Create an OpenRouter account and get an API key and plug it in. Then it will output something like this:
```
Writing to Makefile.secret
# This file is automatically generated by
# build-Makefile.secret.py via the
# Makefile.secret rule in the Makefile.
#
# Do not ever leak this file. If leaked, you will have to
# regenerate it as quickly as you can, before the hackers
# start bombing your deployment.

export DJANGO_SECRET_KEY="cuqt6abl1&o_$dd*4rd(%8(=(=1fmh8@u7g-5laugm$_f!_7!n"
export OPENROUTER_KEY="<sk-or-v1-SECRET-KEY>"
```
You will now have a `Makefile.secret`. This is the file that this repo uses to manage secrets. See the Docker section to see what to do in production.

Next, cd into `find4u` and run `make build`. 

Then, cd into `finder_proj` and, with the `env` virtual environment running, run `python manage.py migrate`.

Then you can cd back into the project root and run `make` to start the web server.

You will notice that no pages have been indexed. To index some pages, you must start the web crawler.

To start the web crawler, cd into `finder_proj` with the `env` virtual environment running and run:

``` sh
python manage.py shell -c "import crawl; crawl.start_crawl('<seedfile>')"
```

Some seedfiles are given.

## Architecture

### Finder

Finder itself is coded as a Django project. The indexed pages are stored in a Django model called `Indexed`. They have a `rank` associated with them which ranks how high they show up in search results. The searching mechanism is quite simple. It uses Django Q-expressions to run a query with the condition that each search term is either in the title, description, or content. If a search term starts with `site:`, the condition is added that the rest of that term must be in the URL. Then it uses `snowballstemmer` to keep only the results that have the stemmed terms.

### Find4U

Find4U is a Next.js app. It uses OpenRouter to prompt `openrouter/free` to answer questions.

### Reverse proxy

Running `make` runs `<path-to-venv-python>/python Makefile.run.py` (the exact path depends on your OS). `Makefile.run.py` is the actual code that runs the server. It starts the Find4U Next.js server and then uses `waitress` to serve a WSGI application. The application forwards all requests with the path starting with `find4u/` to the Next.js server, and if the path dosen't start with `find4u/`, it forwards it to Finder's Django WSGI app.

## Docker

A joint Finder and Find4U image is available at docker.io/tiashdev/finder:latest. This image does not utilize `Makefile.secret`; instead, the environment variables must be set manually. See your `Makefile.secret` for what variables to set or check the one earlier in this document.

To build a Docker image, run:
``` sh
docker build -t <img name> .
```

To run it, run:
``` sh
docker run -p 80:80 -e DJANGO_SECRET_KEY="<your-secret-key>" -e OPENROUTER_KEY=\"<sk-or-v1-SECRET-KEY>\" <img name>
```
